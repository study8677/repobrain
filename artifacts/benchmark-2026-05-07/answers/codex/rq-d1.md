Answer: `Session` first picks an adapter: defaults mount `https://` and `http://`, and longer custom prefixes sort first. `get_adapter` returns the first case-insensitive prefix match. `HTTPAdapter` owns a direct urllib3 `PoolManager` configured by `num_pools`, `maxsize`, and `block`. For each send, it builds pool key inputs from request scheme/host/port plus TLS/cert settings. Then it chooses proxy vs direct: a matching proxy uses a cached per-proxy `ProxyManager` or `SOCKSProxyManager`; no proxy uses the direct `PoolManager`. Both paths call `connection_from_host(..., pool_kwargs=pool_kwargs)`, so urllib3 selects the actual connection pool by host plus TLS/proxy context, then Requests calls `conn.urlopen`.
