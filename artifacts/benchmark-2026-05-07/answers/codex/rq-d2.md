Answer: Requests defaults TLS verification on: `Session.verify` is `"True"`. If `trust_env` is enabled and `verify` is True/None, `REQUESTS_CA_BUNDLE` then `CURL_CA_BUNDLE` can override it. In adapters, `verify=False` sets `"CERT_NONE"`; a string path becomes `ca_certs` or `ca_cert_dir`; otherwise HTTPS uses `DEFAULT_CA_BUNDLE_PATH` and `"CERT_REQUIRED"`. The default bundle is `certs.where()`, and `requests.certs` gets `where` from certifi; certifi is also a project dependency.
