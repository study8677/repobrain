Answer: `fastapi.BackgroundTasks` is a thin subclass of Starlette’s class; `add_task()` delegates to `super().add_task(...)`. Dependency analysis records `BackgroundTasks` parameters, dependency solving creates/injects one shared `BackgroundTasks()` instance, and returns it as `solved_result.background_tasks`. FastAPI attaches that object to the outgoing response via `background=...`, or sets `raw_response.background` when the endpoint returned a `Response`. The ASGI wrapper then awaits `response(scope, receive, send)`. Starlette sends the response body/stream first, then runs `await self.background()`. Starlette’s `BackgroundTasks.__call__` iterates the queued `BackgroundTask`s; async callables are awaited, sync callables run in a threadpool.
